# 仿 json.cn 界面重构与功能增强计划

我将按照您的要求，将 EasyFormatter 重构为类似 json.cn 的现代化界面，并增加 JSON 专用工具箱。

## 1. 后端逻辑扩展
*   **创建 `app/services/json_service.py`**:
    *   实现 JSON 校验 (`validate`)
    *   实现 JSON 美化 (`pretty_print`)，支持自定义缩进和排序
    *   实现 JSON 压缩 (`minify`)
    *   实现 JSON 转义/去转义 (`escape`/`unescape`)
*   **扩展 `app/core/rules.py`**:
    *   虽然主要 JSON 功能通过服务调用，但我会保留将其封装为规则的可能性，以便未来集成。

## 2. UI 布局重构 (`app/ui/main_window.py`)
我们将采用 **Dock栏 + 侧边栏 + 主工作区** 的三栏布局：

### A. 左侧固定 Dock 栏
*   **位置**: 窗口最左侧，宽度固定（约 50-60px）。
*   **组件**:
    *   **"JSON工具" 按钮**: 图标 + 文字，点击切换侧边栏至 JSON 工具页。
    *   **"格式化" 按钮**: 图标 + 文字，点击切换侧边栏至 格式化选项页。
*   **交互**: 点击当前激活的按钮可折叠/展开侧边栏。

### B. 动态侧边栏 (Sidebar)
*   **位置**: Dock 栏右侧，宽度可调 (200-400px)，支持动画展开/收起。
*   **内容 (QStackedWidget)**:
    *   **页面 1: JSON 工具箱**:
        *   功能按钮组：校验、压缩、转义、去转义。
        *   状态显示：显示校验结果（有效/无效及错误信息）。
    *   **页面 2: 格式化选项**:
        *   **JSON 配置**: 缩进下拉框 (2空格/4空格/Tab)、属性排序复选框。
        *   **文本清洗规则**: 迁移现有的“去除空格”、“去除分隔符”等规则复选框到此处。

### C. 主工作区
*   保持现有的 **Header Bar + 左右分栏输入输出** 布局不变，确保核心体验一致。

## 3. 样式与动画 (`app/ui/styles.qss`)
*   **Dock 栏样式**: 深色或对比色背景，按钮悬停/激活高亮。
*   **侧边栏样式**: 与 Dock 栏区分，保持简洁。
*   **动画**: 使用 `QPropertyAnimation` 实现侧边栏宽度的平滑变化。

## 4. 交互逻辑实现
*   **JSON 工具响应**: 点击工具按钮直接处理输入框内容，并更新到输出框（或直接修改输入框，取决于具体功能习惯，通常“压缩/转义”是生成新内容）。
*   **格式化响应**:
    *   当在“格式化”页修改缩进或规则时，实时触发预览（保持现有的防抖机制）。
    *   将 JSON 美化逻辑集成到格式化流程中。

## 5. 测试计划
*   验证 Dock 按钮切换页面是否流畅。
*   验证侧边栏折叠/展开动画。
*   测试 JSON 校验、压缩、转义功能的正确性。
*   确保原有文本清洗规则依然有效。

确认后，我将开始执行上述重构。
